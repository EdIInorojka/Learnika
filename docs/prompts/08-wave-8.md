# Codex prompt — Wave 8 controlled expansion

 

Use this prompt only after the existing product has passed the commercial and educational gates. Do not activate multiple new age bands or subjects in one implementation task.

 

Before code, require for the selected expansion:

 

- approved PRD;

- quantified demand;

- named method owner;

- versioned curriculum package;

- original content plan;

- gold evaluation set;

- age-specific UX research;

- privacy and safety review;

- unit-economics and operations plan;

- migration and rollback plan.

 

Select exactly one expansion package:

 

1. mathematics grades 5-6;

2. mathematics grades 10-11 and EGE;

3. Russian language;

4. primary school;

5. one STEM subject;

6. one foreign language with approved speech scope;

7. preschool ages 3-6;

8. advanced or olympiad product;

9. high-load service extraction.

 

The orchestrator must create focused subagents for product, curriculum, UX, architecture, data, security, QA and independent review.

 

Required process:

 

1. compare the new package with the existing canonical skill and identity platform;

2. identify reusable and non-reusable components;

3. define new domain entities, content schemas and evaluation sets;

4. define age-specific accessibility and parent-control rules;

5. define subject-specific deterministic and AI validation boundaries;

6. define voice, text-to-speech or simulation scope only when relevant;

7. implement the smallest approved vertical slice;

8. run learning, safety, security, performance and economic gates;

9. stop after the selected package; do not begin the next expansion.

 

For preschool or primary products, do not reuse middle-school UX as a visual reskin. For foreign languages, voice data requires a new explicit privacy and evaluation gate. For high-load extraction, provide measured evidence that the modular monolith can no longer meet the required scale or lifecycle.

 

Final verdict must be APPROVE, APPROVE WITH FIXES or BLOCK.
