# Educational content quality

 

## Content types

- explanation;

- worked analogous example;

- exercise template;

- concrete exercise;

- internal solution model;

- hint script;

- error explanation;

- transfer problem;

- diagnostic item;

- assessment item;

- rubric.

 

## Authoring workflow

1. Skill and intended evidence are defined.

2. Author creates original content.

3. Mathematical review checks correctness and edge cases.

4. Pedagogical review checks age, language and cognitive load.

5. Safety review checks answer leakage and inappropriate content.

6. Automated generation tests run for parameterized tasks.

7. Content is published as a version.

8. Production metrics and reports trigger review.

 

## Exercise requirements

Every exercise has:

 

- skill mapping;

- grade and difficulty;

- source and rights metadata;

- expected answer representation;

- deterministic solution model where possible;

- common error taxonomy;

- hint script;

- transfer relation;

- review status;

- publication version.

 

## Hint design

Hint levels should normally progress:

 

1. orient attention;

2. recall a rule or concept;

3. identify the next action;

4. show an analogous example with different data;

5. explain the learner's specific error.

 

A hint must not contain enough information to reconstruct the source final answer by simple copying.

 

## Language rules

- respectful and non-judgmental;

- no labels such as lazy, weak or incapable;

- explain the action, not the personality;

- concise by default;

- define symbols and terms;

- adapt complexity to grade;

- avoid unnecessary motivational pressure.

 

## Quality metrics

- mathematical correctness;

- recognition compatibility;

- item difficulty;

- transfer success;

- hint usefulness;

- answer-leak rate;

- abandonment after hint;

- error-distribution anomalies;

- complaint rate;

- method-review pass rate.

 

## Rollback

Content publication must support rollback by version without losing historical attempt references.
