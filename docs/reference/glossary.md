# Glossary

 

## Product terms

**Student mode** — the learner interface where the original final answer and full solution are hidden.

 

**Parent mode** — the adult interface for consent, payment, goals and progress reports.

 

**Teacher mode** — the interface for specialists supporting individual learners or classes.

 

**School mode** — the organization-aware interface for classes, assignments, assessments and analytics.

 

**Learning track** — a sequence of plan items selected for a goal.

 

**School track** — learning aligned with the selected textbook and current class pace.

 

**Restorative track** — learning selected to close prerequisite gaps.

 

**Target track** — preparation for a test, OGE, EGE or advanced goal.

 

**Transfer problem** — a similar but not identical problem used to verify that the learner can apply the method independently.

 

**Escalation** — a case sent to a mentor, teacher or support specialist because automation is insufficient or confidence is low.

 

**Voice input** — an optional interface that converts a learner's short recording into editable text.

 

**Confirmed transcript** — the text reviewed and explicitly confirmed by the learner; only this text may enter the educational workflow.

 

**Mathematical speech normalization** — conversion of phrases such as “икс в квадрате” into structured mathematical notation while preserving user confirmation.

 

## Curriculum terms

**Skill node** — a measurable knowledge or action that can be assessed.

 

**Skill edge** — a prerequisite or dependency relationship between skills.

 

**Mastery** — a probabilistic state describing current evidence that a learner can use a skill independently.

 

**Curriculum package** — versioned skills, mappings, content, rubrics and evaluation data for an age band and subject.

 

**Textbook mapping** — a versioned relationship between textbook sections and canonical skills.

 

**School pace** — the sequence and dates at which a class studies topics.

 

## Technical terms

**Tenant** — a data boundary such as family, school or organization.

 

**PII** — personally identifiable information.

 

**Gold set** — a reviewed evaluation dataset with expected outputs.

 

**OMR** — optical mark recognition for answer sheets.

 

**ADR** — architecture decision record.

 

**Product gate** — explicit business, methodological, security and technical conditions required before a roadmap stage starts.

 

**VoiceInputSession** — a domain record for one audio recording, transcription, confirmation and deletion lifecycle.

 

**Speech-to-Text adapter** — a provider-neutral interface for transcription services.

 

**Retention deadline** — the timestamp after which temporary raw media must be deleted.
