## Decision policy

- Product scope decisions belong in `product/`.

- Technical trade-offs belong in ADRs.

- Curriculum and mastery decisions belong in `curriculum/`.

- Security exceptions require a written risk acceptance and owner.

- Documents must be updated in the same change as the behavior they describe.

- This Word package is the final bootstrap reference; after files are copied, the Git repository Markdown files become the living source of truth.
